# Proyecto Churn – Modelos y Datasets  
**Soporte para análisis y modelamiento en Machine Learning**

Este directorio contiene los **recursos base** utilizados para el desarrollo del proyecto de predicción de fuga de clientes (*churn*). Aquí se organizan los datasets originales, los modelos entrenados y material de referencia teórica que respalda las decisiones metodológicas adoptadas a lo largo del trabajo.

---

## Contenido del directorio

### `Datasets_base`
Carpeta que contiene los **datasets originales** utilizados en el proyecto.  
Estos archivos corresponden a los datos base empleados en todas las tareas y actividades, y sirven como punto de partida para los procesos de preprocesamiento, entrenamiento y evaluación de modelos.

Características generales:
- Datos de clientes de una empresa de telecomunicaciones.
- Variables numéricas y categóricas.
- Variable objetivo asociada a la fuga de clientes (*churn*).

---

### `Modelos`
Carpeta destinada a almacenar **modelos entrenados**, resultados intermedios o artefactos derivados del proceso de modelamiento.  
Su objetivo es facilitar la reutilización, comparación y análisis posterior de distintos enfoques de aprendizaje automático.

Puede incluir, por ejemplo:
- Modelos guardados.
- Resultados de experimentos.
- Archivos auxiliares generados durante el entrenamiento.

---

### `Paper.pdf`
Documento de referencia teórica utilizado como respaldo conceptual del proyecto.  
Este material aporta fundamentos sobre técnicas de aprendizaje automático, métricas de evaluación y enfoques de modelamiento aplicables al problema de churn.

---

## Consideraciones metodológicas

- Los recursos contenidos en este directorio apoyan el desarrollo práctico del proyecto, pero **no constituyen entregables finales** por sí mismos.
- El uso de un conjunto de datos base común permite mantener coherencia y comparabilidad entre los distintos modelos evaluados.
- La separación entre datos, modelos y material teórico favorece una estructura ordenada y replicable del proyecto.
